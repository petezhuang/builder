## 模块边界和职责

### 🎯 核心边界原则

1. **现有功能保持独立**: 原有的生图和推荐逻辑完全保持不变
2. **新功能非侵入式集成**: 通过依赖注入和事件驱动方式集成
3. **数据层向下兼容**: 扩展现有表结构，不破坏原有数据
4. **服务层清晰分工**: 新增服务专注用户画像相关功能
