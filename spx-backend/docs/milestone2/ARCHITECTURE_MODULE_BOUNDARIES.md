## 核心模块边界定义

### 1. 前端应用层
**职责边界**:
- 用户界面交互
- 预加载图片展示
- 生图和推荐界面
- 用户行为事件收集

**技术栈**: React/Vue + TypeScript

### 2. API网关层  
**职责边界**:
- HTTP路由分发
- 认证授权控制
- 请求限流和监控
- 响应格式统一

**现有框架**: YAP Framework (Go语言)

### 3. Controller层（业务控制层）
**职责边界**:

#### 现有Controller (保持不变)
- `ImageRecommendController`: 图片推荐主流程控制
- `ThemeController`: 主题配置管理  
- `SVGGenController`: AI生图流程控制

#### 新增Controller
- `UserProfileController`: 用户画像管理
- `PreloadController`: 预加载逻辑控制
- `AnalyticsController`: 数据分析和统计

**集成方式**: 现有Controller通过依赖注入调用新增Service

### 4. Service层（业务逻辑层）
**职责边界**:

#### 现有Service (扩展增强)
- `AlgorithmService`: 增加个性化搜索能力
- `SVGGenService`: 增加基于画像的prompt优化

#### 新增Service  
- `ProfileService`: 用户画像计算和管理
- `PreloadMatchingService`: 图库匹配算法
- `CacheService`: 智能缓存管理

**模块间通信**: 通过明确的接口进行服务间调用

### 5. Model层（数据访问层）
**职责边界**:

#### 现有Model (保持兼容)
- `AIResource`: 扩展个性化评分字段
- `User`: 关联用户画像信息

#### 新增Model
- `UserProfile`: 用户画像数据结构
- `UserBehavior`: 行为数据记录
- `PreloadCache`: 预加载缓存管理

**数据一致性**: 通过事务确保跨表操作的数据一致性

### 6. Infrastructure层（基础设施层）
**职责边界**:

#### 现有基础设施 (复用)
- PostgreSQL: 主数据库
- Redis: 缓存和会话管理
- 对象存储: 文件和图片存储

#### 新增基础设施
- 向量数据库: 图片特征向量存储
- 消息队列: 异步任务处理
- 定时任务: 周期性数据处理

## 模块间依赖关系

### 依赖方向 (从上到下)
```
Frontend → API Gateway → Controller → Service → Model → Infrastructure
```

### 关键集成点

1. **现有推荐流程增强**:
   ```
   POST /images/recommend → ImageRecommendController → ProfileService.recordBehavior()
   ```

2. **预加载图片获取**:  
   ```
   GET /user/preload-images → PreloadController → PreloadMatchingService
   ```

3. **用户画像更新**:
   ```
   用户行为 → ProfileService → 异步队列 → 画像重计算
   ```

4. **缓存预热机制**:
   ```
   定时任务 → PreloadController → CacheService → Redis/向量数据库
   ```

## 性能边界和SLA

### 响应时间要求
- 预加载图片API: < 200ms (缓存命中)
- 个性化推荐: < 500ms (现有基础上+100ms)
- 用户画像更新: 异步处理，不影响主流程

### 并发处理能力
- 支持1000+并发用户
- 预加载缓存命中率 > 70%
- 系统可用性 > 99.5%

### 数据存储边界
- 用户画像数据: 50MB/用户
- 预加载缓存: 2GB总容量  
- 行为数据保留: 90天

## 安全边界

### 数据访问控制
- 用户只能访问自己的画像数据
- 管理员接口需要额外权限验证
- 敏感数据加密存储

### API安全策略
- 继承现有的Casdoor认证体系
- API限流: 100请求/分钟/用户
- 数据脱敏: 日志中不记录敏感信息

## 监控边界

### 业务监控
- 用户画像准确率
- 预加载命中率 
- 个性化推荐满意度

### 技术监控  
- API响应时间和错误率
- 数据库性能指标
- 缓存使用情况

### 告警策略
- 系统异常: 5分钟内恢复
- 性能下降: 响应时间超过阈值
- 数据异常: 画像计算失败率 > 5%

这个模块边界设计确保了新功能与现有系统的完美集成，同时保持了清晰的职责分离和高内聚低耦合的架构原则。
