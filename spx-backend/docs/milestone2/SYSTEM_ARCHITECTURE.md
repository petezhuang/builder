# 用户画像与预生成系统架构设计文档

## 1. 概述

用户画像与预生成系统是在现有SPX Builder AI推荐系统基础上构建的智能化扩展，旨在通过用户行为分析和资源预生成，提升系统响应速度和用户体验。

### 2.2 架构分层说明

#### 2.2.1 用户访问层
- **Web UI**: 前端界面，支持用户交互
- **Mobile App**: 移动端应用
- **API SDK**: 第三方集成接口

#### 2.2.2 API网关层
- **认证授权**: JWT token验证，用户权限控制
- **限流熔断**: API请求频率控制，服务降级保护
- **监控日志**: 请求链路追踪，性能监控

#### 2.2.3 业务服务层
- **用户画像服务**: 用户行为分析和画像构建
- **智能推荐服务**: 个性化推荐算法
- **预生成管理服务**: 资源预生成任务调度
- **系统监控服务**: 系统性能监控和告警

#### 2.2.4 AI生成层
- 复用现有的Provider架构
- 支持SVGIO、Recraft、OpenAI三种Provider

#### 2.2.5 数据存储层
- **MySQL**: 主要业务数据存储
- **Redis**: 缓存和会话数据
- **ClickHouse**: 大数据分析存储
- **Kodo**: 图片资源存储

## 4. 数据流设计

### 4.1 用户行为数据流

```
用户交互 → API网关 → 行为收集器 → 消息队列 → 批处理器 → 数据库存储
                     ↓                    ↓           ↓
                实时缓存              画像计算      统计分析
                     ↓                    ↓           ↓
                个性化推荐            画像更新    监控告警
```

**流程说明:**
1. 用户在界面上进行搜索、选择、评分等操作
2. API网关记录请求日志，行为收集器异步处理
3. 实时数据进入消息队列，保证数据不丢失
4. 批处理器定期处理队列数据，更新画像和统计
5. 更新后的画像数据用于个性化推荐

### 4.2 预生成任务数据流

```
需求预测 → 任务生成 → 优先级排序 → 资源分配 → 执行调度 → 质量控制 → 资源存储
    ↓         ↓          ↓          ↓         ↓         ↓         ↓
 历史分析   任务队列    调度算法    成本控制   并发执行   质量评估   缓存更新
```

**流程说明:**
1. 需求预测模块分析用户行为，预测可能的需求
2. 生成预生成任务，加入任务队列
3. 调度器根据优先级和资源约束安排执行
4. 并发执行引擎调用AI Provider生成资源
5. 质量控制器评估结果，合格资源进入存储系统

### 4.3 推荐服务数据流

```
用户请求 → 画像查询 → 召回策略 → 结果排序 → 结果融合 → 用户响应
    ↓         ↓         ↓         ↓         ↓         ↓
 参数解析   偏好获取   多路召回   个性化排序  去重处理   反馈收集
```

**流程说明:**
1. 用户发起推荐请求，系统解析参数
2. 查询用户画像，获取偏好信息
3. 执行多路召回策略（预生成+搜索+实时生成）
4. 基于用户偏好进行个性化排序
5. 融合多路结果，去重后返回给用户
